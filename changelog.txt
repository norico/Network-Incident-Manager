= 2.5.0 =
* Fixed: `incident-scheduled.php` was using CSS class `.nim-resolved-item` instead of `.nim-scheduled-item` — scheduled incidents now render with their correct dashed-border style
* Fixed: `NIM_Ajax::update_status()` — added existence check before UPDATE; success was returned even when the incident ID did not exist
* Fixed: `NIM_Ajax::update_status()` — now fires `do_action('nim_status_changed', $id, $old, $new)` hook on status change
* Fixed: `NIM_Admin::handle_incident_save()` — now also fires `nim_status_changed` hook on status change via the admin form
* Fixed: `GET /list` sort was hardcoded to `created_at DESC` regardless of content type; now defaults to `start_at DESC` and supports `?orderby=` (start_at|created_at|updated_at|severity|status) + `?order=ASC|DESC` params; severity sort uses a weighted CASE expression
* Fixed: `$wpdb->prepare()` conditional removed in `list_incidents()` — WordPress accepts an empty params array since 5.3 (Q5)
* Added: `resolved_at datetime NULL` column on `wp_incidents` — set automatically when an incident transitions to Resolved, cleared when re-opened; migration runs on next page load
* Added: `NIM_DB::update_status()` helper — centralises status update + `resolved_at` logic used by AJAX and REST
* Added: `NIM_DB::get_resolved_incidents()` now selects `description` and sorts by `COALESCE(resolved_at, updated_at)` for reliable resolved ordering
* Added: `incident-resolved.php` now displays `resolved_at` (falls back to `updated_at` via COALESCE for pre-2.5.0 rows)
* Added: REST route `DELETE /wp-json/network-incidents/v1/incidents/{id}` — requires `delete_posts`; returns 204 on success, 404 if not found
* Added: Shortcode `[nim_incidents]` — embeds the full status page inside any post or page; supports `resolved_limit` attribute (default 5)
* Added: `NIM_Helpers::severity_label()` and `NIM_Helpers::status_label()` — translated label maps with literal strings extractable by `wp i18n make-pot`; replaces dynamic `__($variable, $td)` calls in templates and admin list
* Added: `NIM_Helpers::MAX_APP_DEPTH = 10` guard in `apps_options_html()` — prevents infinite recursion on corrupted parent data
* Added: `NIM_DB::get_all_incidents_admin()` and `NIM_DB::get_all_apps_admin()` — SQL moved out of `NIM_Admin` pages into `NIM_DB` (Q1)
* Added: `NIM_DB::table()` and `NIM_DB::apps_table()` helpers — eliminate repeated `$wpdb->prefix . NIM_TABLE` string throughout codebase (Q6)
* Added: `NIM_TD` global constant in bootstrap — single source of truth for the text domain; `NIM_Admin::TD` becomes an alias pointing to `NIM_TD` (Q2)
* Changed: `NIM_Plugin::load_classes()` is now `public static` — `activate()` calls it directly instead of duplicating its own `require_once` list (F5)
* Version bump: `NIM_VERSION` → `2.5.0`

* Added: `NIM_DB` — database install, `maybe_upgrade()`, `drop_legacy_columns()`

* Added: `NIM_Frontend` — rewrite rules, template loader, asset enqueue, `get_template_part()`
* Added: `NIM_Ajax` — `wp_ajax_nim_update_status` handler
* Added: `NIM_REST_API` — REST route registration and callbacks; `sanitize_incident_fields()` shared helper
* Added: `NIM_Admin` — admin menu, all four admin pages, form handlers, asset enqueue
* Fixed: `PRAGMA table_info()` now routed directly to PDO on SQLite instead of going through `$wpdb` (which rejects non-MySQL syntax via the sqlite-database-integration mu-plugin)
* Fixed: `nim_maybe_upgrade()` now flushes rewrite rules after upgrade so `/incidents/` stays reachable without manual reactivation
* Fixed: `nim_api_update()` — existence check now runs before `UPDATE`, not after; returns 404 before touching the database

* Fixed: REST PUT/PATCH route now accepts both methods for broader client compatibility
* Fixed: admin list page — "Date" column now shows `start_at` (incident start) instead of `created_at` (entry creation)
* Fixed: `page_app_edit()` label now reads "Application Name" when editing an existing app (was always "New Application Name")
* Fixed: `$allowed_severity` / `$allowed_status` duplication across 4 functions eliminated — replaced by `NIM_Helpers::SEVERITIES` and `NIM_Helpers::STATUSES`
* Added: plugin header fields `Requires at least: 6.3` and `Requires PHP: 8.1`
* Added: backward-compat shim `nim_get_template_part()` global function so existing theme overrides are not broken
* i18n: `.po` / `.mo` updated — 16 strings added that were missing after v2.3.0 (frontend page, REST API args, AJAX errors)
* Version bump: `NIM_VERSION` → `2.4.0`
