= 2.3.0 =
* Removed: `application varchar(255)` column from `wp_incidents` — unused since 2.1.0 (replaced by `app_id` + `wp_incident_apps`)
* Migration runs automatically on next page load via `nim_drop_legacy_columns()`

= 2.2.0 =
* Added: `start_at` field (datetime) on incidents — default to current time
* Added: "Scheduled" status (replaces "Open") — incidents planned for a future date
* Added: Auto-transition cron (hourly) + page-load check: Scheduled → In Progress when start_at is reached
* Added: "Scheduled Incidents" section on the /incidents/ frontend page
* Added: `incident-scheduled.php` template part
* Removed: "Open" status (renamed to "Scheduled")
* Removed: "Maintenance" status
* Changed: Default incident status is now "Scheduled"
* Changed: Active incidents section now shows start_at instead of created_at

= 2.1.0 =
* Added: `wp_incident_apps` table — hierarchical application management
* Added: "Applications" admin submenu with full CRUD (list, add, edit, delete)
* Added: Parent/child application support (re-parenting on delete)
* Added: `app_id` column on `wp_incidents` replacing the free-text `application` field
* Added: Applications dropdown (indented hierarchy) in incident edit form
* Added: Frontend page at /incidents/ — virtual URL via rewrite rule
* Added: Template override system: theme/nim/{part}.php > plugin fallback
* Added: `nim_get_template_part()` helper function
* Added: `templates/page-incidents.php` — default frontend template
* Added: `templates/parts/incident-active.php` — active incident card
* Added: `templates/parts/incident-resolved.php` — resolved incident row
* Added: `assets/css/frontend.css` — default frontend styles (loaded only without theme override)
* Added: "Recently Resolved" section (last 5) on frontend page
* Added: REST API routes: POST /incidents (create), PUT /incidents/{id} (update)
* Fixed: Timezone handling — all timestamps stored in UTC, displayed via wp_date()

= 2.0.0 =
* Breaking: Removed Custom Post Type `incident` — incidents no longer stored in wp_posts
* Added: Dedicated `wp_incidents` table (reference, description, severity, status, app_id, author_id)
* Added: Custom admin menu "Incidents" with list page and add/edit form
* Added: Inline status update on list page (AJAX, no page reload)
* Added: `assets/js/admin.js` for admin interactions
* Changed: REST API /list now queries wp_incidents directly
* Changed: "Maintenance" status added
* Removed: Gutenberg sidebar (incident-settings-sidebar)
* Removed: `incident_app` taxonomy (replaced by wp_incident_apps in 2.1.0)

= 1.0.0 =
* Initial release
* Custom Post Type `incident` with taxonomy `incident_app` (Applications)
* Custom table `wp_incident_logs` (severity, status per post)
* Gutenberg sidebar panel (ES5) for severity and status fields
* REST API route GET /network-incidents/v1/list
* i18n support with fr_FR translation
